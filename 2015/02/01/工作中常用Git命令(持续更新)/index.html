–no-ff指的是强行关闭fast-forward方式
fast-forward方式就是当条件允许的时候，git直接把HEAD指针指向合并分支的头，完成合并。属于“快进方式”，不过这种情况如果删除分支，则会丢失分支信息。因为在这个过程中没有创建commit
总结来说–no-ff的作用：不使用fast-forward方式合并，保留分支的commit历史
